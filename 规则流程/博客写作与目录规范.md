# 博客写作与目录规范

本仓库博客采用 Hexo 框架,以下规范确保文章能被正确发布、资源路径不会 404。

---

## 一、目录结构规范

所有博客文章统一放在 `source/_posts/` 下,每篇文章配一个**同名资源文件夹**。

### 标准结构示例

```
source/_posts/
├── LazyMondy.md              # 文章 Markdown(在 _posts 顶层,Hexo 才会发布)
└── LazyMondy/                # 与文章同名的资源文件夹
    └── images/               # 资源按类别分子文件夹存放
        └── LazyMondy.jpg     # 该文章用到的图片
```

### 关键规则

1. **文章 md 文件必须放在 `source/_posts/` 顶层**,不能放在子文件夹内。
   - 原因:Hexo 默认只把 `_posts/` 顶层的 `.md` 识别为文章(post),子文件夹内的会被当作普通页面(page),不会出现在首页、归档、分类中。

2. **每篇文章配一个同名资源文件夹**,命名与 md 文件主名一致。
   - 例:`LazyMondy.md` → 资源文件夹 `LazyMondy/`
   - 这是 Hexo `post_asset_folder: true` 的官方约定,构建时资源会被自动复制到文章输出目录。

3. **资源在文件夹内按类别分子文件夹存放**:
   - `images/` — 图片
   - `videos/` — 视频(如有)
   - `files/` — 附件(如有)
   - 未来新增类别按相同方式扩展

4. **资源引用一律使用相对路径**,从 md 文件出发指向资源子文件夹。
   - 例:`![LazyMondy](LazyMondy/images/LazyMondy.jpg)`
5. **每篇文章必须图文并茂** — 正文不能只有纯文字,至少插入一张与内容呼应的插图,放在文章同名资源文件夹的 `images/` 下,用相对路径引用。插图色调与封面图呼应,强化文章气质。

## 二、创建新博客的流程

### 步骤 1:确认配置已开启

在 [_config.yml](file:///d:\Scripts\LazyMondy慢调周记\_config.yml) 中确认:

```yaml
post_asset_folder: true
```

> 若为 `false`,资源相对路径在构建后会 404。

### 步骤 2:创建文章文件与资源文件夹

以文章标题 `MyNewPost` 为例:

1. 在 `source/_posts/` 下创建文章文件:`MyNewPost.md`
2. 在 `source/_posts/` 下创建同名资源文件夹:`MyNewPost/`
3. 在资源文件夹下按需创建分类子文件夹:`MyNewPost/images/`

最终结构:

```
source/_posts/
├── MyNewPost.md
└── MyNewPost/
    └── images/
```

### 步骤 3:放入资源

把文章用到的图片复制到 `MyNewPost/images/` 下。

> 推荐图片命名与文章主名相关,避免使用 `1.jpg`、`2.png` 这类无意义名字。

### 步骤 4:编写文章 Front Matter 与正文

```markdown
---
title: MyNewPost
date: 2026-07-10 12:00:00
tags:
  - 标签1
  - 标签2
---

正文内容...

![图片说明](MyNewPost/images/图片名.jpg)
```

### 步骤 5:本地预览验证

```bash
hexo server
```

访问 `http://localhost:4000` 确认:
- 文章出现在首页列表
- 图片正常显示
- 永久链接可访问

### 步骤 6:提交发布

```bash
git add .
git commit -m "发布新文章: MyNewPost"
git push
```

GitHub Actions 会自动构建并部署到 `gh-pages` 分支。

## 三、封面缩略图策略

本博客首页文章列表采用「缩略图 + 文章信息」横向并排布局,每篇文章都必须配一张**独特封面图**,作为文章的视觉身份标识。

### 设计意图

1. **每篇封面各不相同** — 让读者在首页列表中通过缩略图快速区分不同文章,形成视觉记忆点。
2. **缩略图而非大图** — 封面以小尺寸(桌面端 160×100px,移动端 100×70px)显示在文章卡片左侧,与文章标题、摘要并排。这样可以让用户一眼明白「封面与文章是一体的」,而不是把封面误认为独立的装饰图。
3. **配色贴合文章气质** — 封面图的色调应与文章内容呼应(例如清晨文章用晨光金、地铁文章用隧道蓝灰、自然文章用叶绿),增强内容辨识度。

### 实现方式

- **封面图存放位置**:统一放在 `source/images/` 下,文件名以 `cover-` 前缀加文章主名,例如 `cover-lazymondy.svg`。
- **封面图格式**:推荐使用 SVG 矢量图(体积小、清晰、可主题化),也可使用 jpg/png。
- **Front Matter 配置**:在文章 md 头部通过 `cover` 字段指向封面路径(绝对路径以 `/images/` 开头)。

```markdown
---
title: 文章标题
date: 2026-07-10 12:00:00
cover: /images/cover-文章主名.svg
---
```

- **样式由 CSS 统一控制**:首页缩略图的尺寸、圆角、阴影、flex 布局均在 theme-switch.css 末尾的「封面图缩略图化」段落统一管理,无需每篇文章单独设置。

### 关键规则

1. **每篇文章必须配封面**,且 `cover` 字段指向的文件必须真实存在,否则首页会显示空白或裂图。
2. **每篇封面图必须唯一**,不允许两篇文章共用同一张封面。
3. **封面文件名以 `cover-` 开头**,与文章主名关联,方便检索维护。
4. **不在 `_config.butterfly.yml` 配置 `default_cover`** — 因每篇文章都有专属封面,默认封面会削弱「每篇独特」的视觉意图,故留空。
5. **新文章创建流程**:
   - 先在 `source/images/` 下制作 `cover-新文章主名.svg`(或 jpg/png)
   - 在文章 Front Matter 中写入 `cover: /images/cover-新文章主名.svg`

## 四、Front Matter 模板

```markdown
---
title: 文章标题            # 必填
date: 2026-07-10 12:00:00  # 必填,格式 YYYY-MM-DD HH:mm:ss
tags:                      # 可选
  - 标签1
  - 标签2
categories:                # 可选
  - 分类名
description: 文章描述       # 可选
cover: /images/cover-文章主名.svg  # 必填,每篇封面必须唯一
---

正文第一段...

![插图说明](文章主名/images/插图名.svg)  # 必填,至少一张内文插图

正文继续...
```

## 五、常见错误与规避

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| md 文件放在子文件夹内 | 文章不发布 | md 必须在 `_posts/` 顶层 |
| 资源文件夹与文章不同名 | 相对路径失效 | 文件夹名 = md 主名 |
| `post_asset_folder: false` | 构建后图片 404 | 配置为 `true` |
| 用绝对路径引用资源 | 部署后失效 | 用相对路径 `文章名/分类/文件` |
| 图片放在 `_posts/` 顶层 | 混乱难维护 | 放进文章资源文件夹的 `images/` |
| 文章未配 cover 字段 | 首页缩略图位置空白 | 每篇必须配 `cover: /images/cover-xxx.svg` |
| 两篇文章共用同一封面 | 失去视觉辨识度 | 每篇封面必须唯一,文件名以 `cover-` 前缀区分 |
| cover 路径指向不存在的文件 | 首页显示裂图 | 写入前确认 `source/images/` 下文件已存在 |
| 在 butterfly 配置 default_cover | 削弱「每篇独特」意图 | 留空,每篇用专属 cover |
| 文章正文无任何插图 | 纯文字枯燥,缺乏图文呼应 | 至少插入一张插图,放在 `文章名/images/` 下 |
